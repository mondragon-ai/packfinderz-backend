# Title [PF-287]: Create `reviews` table migration (Goose) + DB types

## Type

Infra

## Description

Add the `reviews` table (and required enum/type dependencies) via Goose migration so the backend can persist store reviews (buyer writes review for vendor store).

## Scope

* What is included

  * Goose up/down migration to create `reviews` table
  * Create/ensure `review_type` enum exists (or equivalent) used by `reviews.review_type`
  * Add indexes needed for primary access paths (e.g., `vendor_store_id`, `created_at`, optionally `(vendor_store_id, created_at)` for pagination)
  * Ensure FK constraints match the intended delete behaviors
* What is explicitly NOT included

  * Any API/controller/service logic
  * Any UI/public store GET join work

## Acceptance Criteria

* [ ] Running Goose up creates the `reviews` table with all columns, constraints, and FK behaviors exactly as defined in Master Context
* [ ] Running Goose down removes the table (and only removes enum/type if it was introduced in the same migration and is safe to drop)

## Dependencies

* Blocked by:

  * None
* Blocks:

  * PF-XXX Repo implementation
  * PF-XXX Service + CRUD API

## Technical Notes

* Relevant services, tables, APIs, queues, or workers

  * Table: `reviews`
  * Referenced tables: `stores`, `users`, `products`, `orders`
  * Ensure `gen_random_uuid()` availability (pgcrypto/uuid extension) is already handled elsewhere; otherwise add it

  ```SQL
    CREATE TABLE reviews ( 
        id UUID PRIMARY KEY DEFAULT gen_random_uuid(), 
        review_type review_type NOT NULL, 
        
        -- Who wrote it 
        buyer_store_id UUID NOT NULL REFERENCES stores(id) ON DELETE CASCADE, 
        buyer_user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, 
        
        -- What is being reviewed 
        vendor_store_id UUID REFERENCES stores(id) ON DELETE CASCADE, 
        
        -- Optional integrity enforcement 
        product_id UUID REFERENCES products(id) ON DELETE CASCADE, 
        order_id UUID REFERENCES orders(id) ON DELETE SET NULL, 
        
        rating SMALLINT NOT NULL CHECK (rating >= 1 AND rating <= 5), 
        title VARCHAR(150), 
        body TEXT, 
        
        is_verified_purchase BOOLEAN DEFAULT false, 
        is_visible BOOLEAN DEFAULT true, 
        
        created_at TIMESTAMP NOT NULL DEFAULT NOW(), 
        updated_at TIMESTAMP NOT NULL DEFAULT NOW(),
    );
  ```


* Assumptions (explicitly marked)

  * **ASSUMPTION**: `review_type` enum already exists; if not, introduce it in this migration.

## Out of Scope

* Product reviews behavior beyond optional `product_id` FK
* Verified purchase logic (service-layer validation)
